博客初搭建

发表于2024-05-03|更新于2024-05-05
|字数总计:1.3k|阅读时长:4分钟|阅读量:

前言

一时心血来潮，准备搭建个人网站。其实我数年前是有记录东西的习惯的，可随着时间的推移，记的东西逐渐丢失，无论是纸质的还是互联网平台上的。而个人网站却是能寿命悠远——当然啦，网站代码肯定得好好保管，多多备份。

除此之外，日渐觉得记忆力靠不住，尤其是学到的知识，学完就忘，总觉得可惜了辛苦组织的知识结构，真不如找个媒介记录下。于是，我想到的是博客。它的图文形式饱含信息量，可以让我最短时间内复习完毕，同时书写的过程更是能加深对知识的理解，何乐而不为呢？

于是，我选择了这个存储简单、方便修改、图文结合、多端访问的“老古董”，博客。

搭建

网站的搭建，对于新手的我确实存在困难。我当然也想过用wordpress这样的集成度很高上手容易的方案，但是，我的一些想法是需要足够高的自定义空间的。因此，我会选择更加“折腾”的方案。Hexo这个快速、简洁且高效的博客框架便进入了我的视野。它使用 Markdown（或其他标记语言）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

借助B站教程我成功的将博客框架上传至Github仓库托管。

此处记录一下常用代码

生成静态文件：

hexo g

本地内容预览：

hexo s

静态网页部署：

hexo d

同样，选择观看B站教程。

Butterfly安装方法

在你的博客根目录里

git clone -b master https://github.com/jerryc127/hexo-theme-butterfly.git themes/Butterfly

如果想要安装比较新的dev分支，可以

git clone -b dev https://github.com/jerryc127/hexo-theme-butterfly.git themes/Butterfly

使用mermaid标签可以绘制Flowchart（流程图）、Sequence diagram（时序图 ）、Class Diagram（类别图）、State Diagram（状态图）、Gantt（甘特图）和Pie Chart（圆形图），具体可以查看mermaid文档

还有内嵌图片、时间线、乐谱、系列文章等多个功能。

为了使用数学公式，我使用了以下命令：

$ npm un hexo-renderer-marked --save

$ npm i hexo-renderer-markdown-it --save

卸载掉 marked 插件，安装 hexo-renderer-markdown-it，这样就可以使用更快更轻量的KaTeX来渲染数学公式啦

不过要注意，须在使用的Markdown Front-matter 加上 katex: true
